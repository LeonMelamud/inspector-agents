### Step 1: Set Up Your Account
1. Go to https://dev.to/ and create an account
2. Complete your profile (bio, profile image, social links)
3. Link your GitHub account for credibility
4. Mention you're from InspectAgents.com

### Step 2: Create a New Post
1. Click "Write a post" in the top right
2. Dev.to supports markdown with frontmatter
3. Copy-paste the entire article content (including frontmatter)
4. The frontmatter will auto-populate fields

### Step 3: Configure Post Settings
**Important Settings:**
- **Published:** Set to `false` initially (save as draft)

- **Series:** Set to "AI Safety" to link all 3 articles together

### Step 4: Preview & Publish
1. Click "Preview" to see how it looks
2. Check that all markdown renders correctly
3. Verify canonical URL is present
4. Change `published: false` to `published: true`
5. Click "Publish" üéâ

## üîß Troubleshooting

### Issue: Canonical URL Not Showing
**Solution:** Some platforms hide canonical URLs by default. Make sure:
- Frontmatter includes `canonical_url` field
- URL is absolute (starts with https://)
- You've saved the post (some platforms only show on save)

### Issue: Cover Image Not Displaying
**Solution:**
- Use direct image URLs (not embedded images)
- Ensure URL starts with https://
- Try Unsplash URLs provided in frontmatter
- Max recommended size: 1000px wide, <500KB

### Issue: Markdown Not Rendering
**Solution:**
- Check for special characters that need escaping
- Ensure code blocks use triple backticks (```)
- Preview before publishing
- Some platforms have markdown quirks (test in draft mode)

### Issue: Low Engagement
**Solution:**
- Post at optimal times (Tuesday-Thursday 9-11 AM EST)
- Engage in comments within first hour
- Share on Twitter/LinkedIn immediately after publishing
- Ask a question in the article to spark discussion
- Cross-promote in relevant communities
